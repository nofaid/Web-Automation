# Test Cases

## TC-001. Успешная регистрация нового пользователя

### Предусловия

- Пользователь не авторизован.
- Используется новый email, который ранее не был зарегистрирован в системе.

### Шаги

1. Открыть страницу регистрации / входа.
2. Перейти в блок создания нового аккаунта.
3. Ввести имя.
4. Ввести новый email.
5. Нажать кнопку создания аккаунта.
6. Заполнить обязательные поля формы регистрации валидными данными.
7. Завершить регистрацию.

### Ожидаемый результат

Новый аккаунт успешно создаётся. Пользователь получает подтверждение успешной регистрации и может работать в системе как авторизованный.

---

## TC-002. Повторная регистрация на уже существующий email

### Предусловия

- В системе уже существует аккаунт с указанным email.
- Пользователь не авторизован.

### Шаги

1. Открыть страницу регистрации / входа.
2. Перейти в блок создания нового аккаунта.
3. Ввести имя.
4. Ввести email, который уже зарегистрирован в системе.
5. Нажать кнопку создания аккаунта / продолжить регистрацию.

### Ожидаемый результат

Система не позволяет повторно зарегистрировать аккаунт на уже существующий email и отображает понятное сообщение об ошибке.

## TC-003. Проверка обязательных полей формы регистрации

### Предусловия

- Пользователь не авторизован.
- Открыта форма регистрации.

### Шаги

1. Открыть форму регистрации.
2. Оставить одно или несколько обязательных полей пустыми.
3. Попытаться завершить регистрацию.

Система не позволяет завершить регистрацию при незаполненных обязательных полях и показывает ошибку / подсказку по этим полям.

## TC-004. Проверка валидации поля First Name на числовые значения

### Предусловия

- Пользователь не авторизован.
- Открыта форма регистрации.

### Шаги

1. В поле `First Name` ввести числовое значение, например `12345`.
2. Остальные обязательные поля заполнить валидными данными.
3. Завершить регистрацию.

### Ожидаемый результат

Система должна отклонить явно некорректное значение в поле `First Name` и не позволить завершить регистрацию без исправления.

## TC-011. Удаление аккаунта должно требовать подтверждения действия

### Предусловия

- Пользователь зарегистрирован и авторизован в системе.

### Шаги

1. Войти в аккаунт.
2. Нажать кнопку `Delete Account`.

### Ожидаемый результат

Система не должна удалять аккаунт мгновенно. Перед удалением должен появляться шаг подтверждения действия.

---

## TC-012. Поведение системы после успешной регистрации

### Предусловия

- Пользователь не авторизован.
- Используется новый email.

### Шаги

1. Пройти полный сценарий регистрации нового аккаунта.
2. После завершения регистрации обновить страницу браузера.

### Ожидаемый результат

Если по логике продукта пользователь автоматически авторизуется после регистрации, его авторизованное состояние должно сохраняться после обновления страницы без ошибок и противоречий в интерфейсе.
