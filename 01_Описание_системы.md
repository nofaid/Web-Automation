# Описание системы

## 1. Что это за система

Система представляет собой web-интерфейс интернет-магазина.

Пользователь может:

- зарегистрироваться;
- войти в аккаунт;
- просматривать товары;
- открывать карточки товаров;
- добавлять товары в корзину;
- управлять своей покупательской активностью через интерфейс сайта.

## 2. Какой тип продукта это с точки зрения QA

Это e-commerce продукт, то есть система, в которой качество напрямую влияет на:

- удобство пользователя;
- возможность завершить покупку;
- корректность пользовательского сценария;
- риск потери заказа;
- риск потери денег;
- доверие к продукту.

## 3. Какие сущности важны для тестирования

С точки зрения QA в системе важны следующие сущности:

- пользователь;
- аккаунт;
- товар;
- корзина;
- заказ;
- пользовательская сессия;
- состояние авторизации.

## 4. Какие пользовательские сценарии выглядят ключевыми

Ключевые сценарии:

- регистрация нового пользователя;
- вход существующего пользователя;
- просмотр каталога;
- открытие карточки товара;
- добавление товара в корзину;
- переход пользователя по основному пути интернет-магазина.
