# Jira mini-layer

## Цель

Перенести подтверждённые дефекты из Web UI проекта в формат issue tracking system, приближенный к реальной командной работе.

## Какие дефекты перенесены в Jira

- BUG-001. Поле First Name принимает числовые значения
- BUG-002. Поле Last Name принимает числовые и специальные символы без явной валидации
- BUG-003. Поле Mobile Number принимает буквенные символы
- BUG-004. Удаление аккаунта выполняется без шага подтверждения действия

## Что должен показать этот слой

Этот слой демонстрирует, что я умею:

- переносить defect из markdown-описания в issue tracking system;
- правильно формулировать summary;
- оформлять steps to reproduce;
- описывать expected result и actual result;
- указывать priority и severity;
- прикладывать evidence;
- вести defect в рабочем формате, приближенном к Jira-процессу.
