# Вопросы к логике и checklist

## 1. Вопросы к логике перед тестированием

### 1.1 Регистрация

- Можно ли зарегистрироваться на уже существующий email?
- Какие поля обязательны?
- Есть ли ограничения на формат имени, email и пароля?
- Что считается успешной регистрацией?
- Должен ли пользователь автоматически авторизоваться после регистрации?
- Что происходит, если пользователь прерывает заполнение формы?
- Как система показывает ошибку при невалидных данных?

### 1.2 Вход в аккаунт

- Что происходит при неверном email?
- Что происходит при неверном пароле?
- Есть ли ограничение на число попыток входа?
- Что считается успешным входом?
- Как пользователь понимает, что он вошёл в систему?
- Что происходит, если пользователь уже авторизован и снова открывает страницу логина?

### 1.3 Каталог товаров

- Все ли товары отображаются корректно?
- Есть ли у товара обязательные атрибуты на карточке списка?
- Что происходит при переходе к товару?
- Может ли пользователь открыть карточку товара из каталога без ошибки?

### 1.4 Карточка товара

- Достаточно ли информации о товаре для пользователя?
- Есть ли цена, название, описание, изображение?
- Можно ли добавить товар в корзину именно из карточки?
- Как система реагирует после добавления?

### 1.5 Корзина

- Добавляется ли товар в корзину с первого действия?
- Сохраняется ли корзина после перехода между страницами?
- Что видит пользователь после добавления товара?
- Можно ли добавить один и тот же товар повторно?
- Как отображается количество и состав корзины?

## 2. Checklist первой проверки

### 2.1 Регистрация

- [ ] Открывается страница регистрации
- [ ] Видны поля регистрации
- [ ] Успешная регистрация с валидными данными
- [ ] Ошибка при попытке регистрации с уже существующим email
- [ ] Проверка обязательных полей
- [ ] Проверка пустых значений
- [ ] Проверка невалидного email
- [ ] Проверка минимально разумных значений в полях
- [ ] Проверка слишком длинных значений
- [ ] Понятность сообщений об ошибке

### 2.2 Вход

- [ ] Открывается форма входа
- [ ] Успешный вход с валидными данными
- [ ] Ошибка при неверном пароле
- [ ] Ошибка при неверном email
- [ ] Проверка пустых полей
- [ ] Проверка поведения после успешного входа
- [ ] Проверка, что пользователь визуально понимает факт авторизации

### 2.3 Каталог товаров

- [ ] Открывается страница товаров
- [ ] Отображается список товаров
- [ ] У товаров есть основные элементы отображения
- [ ] Переход в карточку товара работает
- [ ] Нет явных поломок отображения списка
- [ ] Список товаров визуально и логически читаем

### 2.4 Карточка товара

- [ ] Карточка открывается без ошибки
- [ ] Видно название товара
- [ ] Видна цена товара
- [ ] Видно изображение товара
- [ ] Видна базовая информация о товаре
- [ ] Доступно добавление в корзину
- [ ] После добавления пользователь получает понятную реакцию системы

### 2.5 Корзина

- [ ] Товар добавляется в корзину
- [ ] Корзина открывается
- [ ] В корзине отображается добавленный товар
- [ ] Количество товара отображается корректно
- [ ] Название товара отображается корректно
- [ ] Цена товара отображается корректно
- [ ] Состояние корзины сохраняется после переходов по страницам
- [ ] Проверка повторного добавления товара
- [ ] Нет явных противоречий между действием пользователя и состоянием корзины

---

## 3. Приоритет первой проверки

### Высокий приоритет

- регистрация
- вход
- добавление товара в корзину
- отображение корзины

### Средний приоритет

- карточка товара
- целостность каталога

### Низкий приоритет на первом проходе

- второстепенные косметические замечания
- мелкие визуальные шероховатости, не влияющие на основной путь пользователя
